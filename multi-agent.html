Phase 02 ／ AIの進化の系譜

機械学習／深層学習
データから特徴を掴む鑑定士。

人間がルールを書くのではなく、AIが大量のデータを観察し、自ら「パターン（特徴）」を見つけ出す。「予測・分類」の精度が劇的に上がった時代。

Concept
In one sentence

何千枚もの実物を見てきた骨董品の鑑定士。
"本物" の特徴を、データから掴んでいる。

Diagram 01

仕組み — 「学習」と「予測」の二段構え

機械学習は2つのフェーズに分かれる。膨大なデータからモデルを作る「学習フェーズ」と、そのモデルを使って未知のものを判定する「予測フェーズ」。

Input ／ 学習データ
大量のラベル付きデータ
過去の例（猫の写真1万枚 + 犬の写真1万枚など）。「正解」と一緒に与える。
→
Training ／ 学習
特徴量を自動抽出する
「猫には三角の耳と細長い目」といった特徴を、データから自分で見つけて重み付けしていく。
→
Output ／ モデル
学習済みモデル
新しい未知の画像を入れると、確率的に「これは猫」と返してくれる関数。

機械学習と深層学習の違い — 「層」の深さ

深層学習（Deep Learning）は機械学習の一種。中間層を何層にも深く重ねることで、より複雑なパターンを掴めるようになった。画像認識のブレイクスルーはここから生まれた。

Machine Learning

機械学習

人間が「目の幅」「耳の角度」などの特徴量を設計し、AIに与える。AIはその範囲内でパターンを学ぶ。

Deep Learning

深層学習

多層の構造により、特徴量そのものをAIが自動で見つけ出す。人間が想像もしなかった特徴を発見することもある。

In practice

業務での代表例

「予測・分類」が機械学習の主戦場。今でも多くの業務システムの裏で動いている、現役のAI。

01

迷惑メールの自動振り分け

過去の迷惑メール／通常メールを大量に学習し、新着メールの「迷惑度」を確率で判定。ルールでは絶対に捌けない、語彙の揺れにも対応する。

02

画像認識（製造業の外観検査）

「不良品の画像」を数千枚学習させると、新しい製品画像から目視レベルの不良を検出できる。ベテラン検査員の代替・補助として広く普及。

03

需要予測・離反予測

「過去の売上、天気、曜日、イベント情報」から来週の需要を予測。「過去の顧客行動」から解約しそうな顧客をスコアリング。

機械学習は 「過去のパターンから、未来を予測する」 AI。
ただし、過去に存在しないものは生み出せない。
Limits

限界 — なぜ「生成AI」が次の段階だったのか

機械学習は強力だが、本質的には「分類器・予測器」。文章を一から書いたり、新しい絵を描いたりすることは不得意だった。

Strength
得意なこと
  • 分類（カテゴリ分け、迷惑メール判定）
  • 予測（売上、需要、故障時期）
  • 異常検知（普段と違う動きの検出）
  • レコメンド（似たユーザーが買った商品）
Weakness
苦手なこと
  • 大量の高品質ラベル付きデータが必要
  • 「なぜそう判断したか」が説明しにくい（ブラックボックス）
  • 学習データに含まれないパターンに弱い
  • 新しいものを「生成」することはできない
Plain Language

用語解説 — むずかしい言葉をやさしく

このページに出てくる専門用語を、業務や日常に置き換えて理解する。

機械学習 Machine Learning

大量のデータから「答えのパターン」をコンピュータ自身に見つけさせる仕組み。人間が「ルール」を書く代わりに、AIが「例題」を解いてコツを掴む。

深層学習 Deep Learning

機械学習の中でも「人間の脳の神経のような仕組み」を何層も重ねて使う方式。画像や音声に強い。伝言ゲームを100段にしたような構造。各段で少しずつ特徴が抽出される。

教師あり学習 Supervised Learning

「正解付きの例題」を大量に与えて、AIに正解を当てさせる訓練方法。問題集と解答集をセットで与え、AIが模試で点数を上げていくイメージ。

パラメータ Parameter

学習を通じて自動調整される、AIの中の "つまみ" のような数値。ラジオのチューニングのつまみ。何億個ものつまみを少しずつ回して、最適な周波数に合わせていく。

過学習 Overfitting

訓練データばかり完璧に覚え、新しい問題が解けなくなる状態。過去問の答えを丸暗記して、ちょっと問題文が違うだけで解けない受験生。

特徴量 Feature

AIが判断のために注目する「手がかり」のこと。猫の写真を見分けるとき、「耳の形」や「ヒゲ」に注目する、その注目ポイント。
