Session 08 ／ エージェントチーム

エージェント同士の
議論プロトコル
"合議" のフォーマット。

複数のAIに合議させるには、ただ並べるだけでは足りない。意見表明 → 批判 → 修正 → 合意 — どの順番で、誰が発言し、いつ終わるかを決めるルールが必要。

In one sentence

AIチームに "議事ルール" を渡す。
誰がいつ口を開き、誰が決を取るかを決めておく。

Diagram 01

議論プロトコルの基本フロー — 4ステップ

AI同士が無秩序に会話すると、堂々巡りや延々と続くループに陥る。"ステップ＋上限回数" を設計に組み込む。

01Propose
意見表明 ／ 専門家が最初の案を提示
指揮役からの依頼を受け、各専門家エージェントが「私の案」を提出する。並列で進める。
02Critique
批判 ／ 批評役が問題点・盲点を指摘
「この案は〇〇のリスクがある」「別視点ではこう見える」と、批評エージェントが懐疑的に突っ込む。複数の批評役を置いてもよい。
03Revise
修正 ／ 提案者が批判を受けて改稿
最初の専門家が、批判を踏まえて案を更新する。修正→批判のループを 1〜2 回まで許す（無制限にしない）。
04Synthesize
統合 ／ オーケストレーターが結論をまとめる
最終案、残った論点、それぞれの根拠を、人間が判断できる形に整理して提示。"AI同士の議論" を 1 枚のレポートに圧縮する。
Case

サンプル — "新規顧客の与信判断" を AI で合議させる

実際にこのプロトコルが回ると、どんな対話が生まれるか。AIたちの議論は、後から人間が議事録のように読める。

Agent A ／ 財務
過去3年の財務指標は良好。流動比率も健全。→ 与信OKと判断します。
Agent B ／ 批評
ただし、その3年は業界全体の好況期と重なっています。景気減速時のレジリエンスは未検証では？
Agent C ／ 営業履歴
過去にうちの顧客で2社、同社グループから倒産案件あり。グループ全体の信用情報の確認が必要そうです。
Agent B ／ 批評
グループ会社の与信情報を取得した上で、再評価すべき。"条件付き承認" が妥当では？
Orchestrator
3エージェントの議論を統合した結果、条件付き承認を推奨。条件は ① グループ会社の信用照会、② 与信限度額の段階引き上げ、の2点。人間の最終判断を仰ぐ。
Patterns

議論プロトコルの代表的パターン

問題の性質によって、使うフォーマットを変える。万能はない。

Pattern 01
Round Robin ／ 順番発言型

専門家が順番に意見を述べる。シンプルで予測しやすい。発言順による先行有利／不利が出るので、順番をランダムにすることもある。

向いている：構造的タスクの分担
Pattern 02
Critic-in-the-Loop ／ 批評ループ型

提案 → 批判 → 修正、を 1〜2 回繰り返す。品質を上げるのに有効だが、ループ上限を必ず設定する（無限に続く危険）。

向いている：意思決定・文章レビュー
Pattern 03
Debate ／ 賛否ディベート型

「賛成役」「反対役」を割り当て、それぞれが根拠を提示。第三のエージェントが審判として判定する。盲点が見つかりやすい。

向いている：曖昧な戦略判断
Pattern 04
Voting ／ 投票合議型

複数のエージェントに独立に判断させ、多数決で結論を出す。"アンサンブル" 効果でハルシネーションが減る。

向いている：分類タスクの精度向上
AI同士の議論は "会議のフォーマット" 次第。
ルールがなければ、知能を並べても賢くはならない。
Plain Language

用語解説 — むずかしい言葉をやさしく

このページに出てくる専門用語を、業務や日常に置き換えて理解する。

議論プロトコル Discussion Protocol

AI同士に "合議" させるための手順書。誰が、いつ、何を言うかを規定する。会議の議事次第。"まず A 部長から、次に B 課長、最後に決を取る" のような流れ。

Critic-in-the-Loop CITL

"批評役" を必ずプロセスに組み込むパターン。AIに自己批判させて品質を上げる手法。提案資料を上司レビューに通すフロー。出した瞬間にチェックが入る前提で書く。

アンサンブル Ensemble

複数のAIに独立に判断させ、結果を統合する手法。多数決で精度が上がる。3人の医師に "セカンドオピニオン" を取って、過半数が同じ診断ならその答えを採用、という考え方。

ループ上限 Max Iterations

議論や修正の繰り返し回数の上限。これを設定しないと AI 同士が延々と話し続けてしまう。"最大2回までやり直し" の社内ルール。コスト管理と打ち切り判断のために必須。

合意形成 Consensus

複数の異なる意見を持つAIが、最終的に1つの結論にまとめるプロセス。役員会の "合議"。全員一致じゃなくても、議論を経た上で判断が記録される。

ディベート Debate Protocol

2 体に "賛成" と "反対" を割り当てて議論させる手法。第三者が審判する。学校のディベート大会。立場を強制することで論点が明確になる。
