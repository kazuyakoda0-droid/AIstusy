01
Chapter One

AI活用の基礎と、個人の業務効率化。

まずは「AIを正しく理解する」ことから。AIの進化の歴史、LLMの仕組み、そして個人の生産性を底上げする実践テクニックまで。明日からデスクで使える成功体験を作る章です。

Session 01
Foundation

AIの進化と最終ゴール
— 生成AIから「エージェントチーム」が動く組織へ

目的

研修全体のロードマップを共有し、AIの進化の歴史と、目指すべき組織の姿（ゴール）を解像度高く理解する。

AIの進化
Phase 01
ルールベース
条件分岐する電卓。決められた手順を粛々と実行する。
Phase 02
機械学習 / 深層学習
データを読み込み特徴を掴む鑑定士。予測・分類が得意。
Phase 03
生成AI
文脈を理解しゼロから生み出すクリエイター。
構成案
「予測・分類」から「生成」へ
迷惑メールの分類から、返信メールのドラフト作成へ。従来AIとChatGPT以降のAIの決定的な違い。
AIの進化の先
一問一答のチャットボットから、自律的に動く「AIエージェントチーム」へのパラダイムシフト。
具体例
業務の進化イメージ
分類・抽出 「資料の誤字を探して」
生成 「このメモからプレゼン資料の構成を作って」
エージェント 「〇〇の件、関係者にヒアリングして資料にまとめておいて」
キーワード
機械学習 深層学習 生成AI パラダイムシフト AIエージェント
Session 02
Foundation

LLMの仕組みと「プロンプト」の作法

目的

AIが言葉を紡ぐ仕組みをブラックボックスから解放し、「超優秀だが融通の利かない新入社員」を動かすための適切な指示（プロンプト）構造を習得する。

構成案
LLMとは何か
「次に来る確率が高い単語」を予測する確率モデル。AIは意味を理解しているのではなく、確率的に尤もらしい言葉を紡いでいる。
LLMの得意・不得意
要約・翻訳・アイデア出しが得意な反面、事実確認や複雑な計算が苦手。ハルシネーション（もっともらしい嘘）が起こる原理を理解する。
プロンプトエンジニアリングの基本（4要素）
  • 役割（Role）
  • 文脈・背景（Context）
  • タスク（Task）
  • 出力形式（Format）
Before / After
プロンプト比較
新商品のキャッチコピー作成
Bad ／ 曖昧な丸投げ 新商品のキャッチコピー考えて
Good ／ 構造化された指示 あなたはベテランのコピーライターです（役割）。20代女性向けの時短スキンケアコスメを発売します（文脈）。商品の魅力を伝えるキャッチコピーを（タスク）、箇条書きで5つ、それぞれ解説付きで（出力形式）提案してください。
キーワード
LLM ハルシネーション プロンプトエンジニアリング 構造化プロンプト
Session 03
Practice

個人業務のAIハック
— 文書作成・要約・データ処理の半自動化

目的

明日から各自のデスクですぐに使えるAI活用術を学び、個人の生産性を劇的に底上げする。まずは個人の成功体験を作る。

構成案
文書作成の高速化
箇条書きの雑なメモから、社外向けの丁寧なビジネスメールや報告書のドラフトを生成。
情報圧縮と要約
長大な会議の音声文字起こしやPDF資料から、重要ポイントやTodoを瞬時に抽出。
データ処理・Excel連携
「こういう集計がしたい」という自然言語から、複雑なExcel関数やマクロ（VBA）を生成。
アイデア出しの壁打ち
企画や課題解決における、思考のパートナーとしての活用法。
ライブデモ
推奨デモシナリオ
議事要約
乱雑な文字起こしの一発整理
「決定事項、未決事項、Next Action（誰が・いつまでに・何をするか）を表形式で抽出して」と指示し、一瞬で整理される様子を実演。
壁打ち / 逆引き
AIにファシリテートさせる
「新企画のターゲット層で悩んでいる。私に3つ質問をして、ターゲットを絞り込ませて」とAIに会話を主導させる逆引きプロンプトの紹介。
キーワード

自社データ・社内システムとの連携。

一般的なAIは「自社のこと」を知りません。社内の知識をセキュアに組み込み、既存ワークフローに組み込むための仕組みを学びます。

Session 04
Integration

社内独自の知識をAIに持たせる
— セキュアな環境とRAGの基礎

目的

一般的なAIが知らない「自社の知識」を安全に組み込み、業務特化型の回答を得る仕組みを理解する。

構成案
汎用LLMの限界
「うちの会社の出張規定はどうなってる？」と聞いても答えられない（あるいは嘘をつく）理由。
RAG（検索拡張生成）とは
AIに「オープンブックテスト（持ち込み可のテスト）」を受けさせる仕組み。質問に対し、まず社内マニュアルを検索し、その該当箇所（カンペ）をAIに読ませて回答させる。
セキュリティとローカル環境
機密データを扱う際の注意点。オンプレミスやセキュアクラウドでのAI運用の重要性（オプトアウトの設定など）。
社内ナレッジ共有の未来
属人化していた「あの人に聞かないと分からない」ノウハウを、AI経由で全社員が24時間引き出せる環境へ。
具体例
社内RAGボットへの質問例
User 来月から産休に入ります。必要な手続きを教えて。
RAG-Bot 当社の就業規則第〇条に基づき、以下の手続きが必要です。
1. 〇〇フォーム（URL）の提出
2. 直属上長への通知
3. 人事部への書類提出
出典：社内ポータル / 人事規定集2024年版
キーワード
RAG ベクトルデータベース 社内ナレッジ セキュア環境 オプトアウト
Session 05
Integration

業務ワークフローの自動化
— 既存ツールとAIのシームレスな連携

目的

AIを「チャット画面」から解放し、既存の業務システムやワークフローに組み込んでプロセス全体を自動化する概念を学ぶ。

構成案
「対話型AI」から「組み込み型AI」へ
人間がいちいちコピペする「つなぎ込み」作業をなくすAPI連携の基礎。
人間とAIの分業
AIは「情報の仕分けと下準備」を行い、人間は「最終判断と承認ボタンを押すこと」に集中する。
ケーススタディ
書類の承認（見積・稟議プロセス）
現状の課題：書類のPDFを目視確認、手作業でのシステム入力、担当部署へのメール連絡。
Step 1
RPAやZapier/Make等がメール添付の見積書（PDF）を検知。
Step 2
AIがPDFから「会社名」「品目」「金額」を自動抽出（構造化データ化）。
Step 3
AIが社内規定（RAG）と照合し、「100万円以上なので部長決裁」とルートを自動判定。
Step 4
ワークフローツールへ自動入力し、部長のSlackに承認リクエストを通知。
キーワード
ワークフロー自動化 API連携 RPA 自動ルーティング

自律型AIと、開発の民主化。

「指示待ち」から「自律型」へ。大枠のゴールを与えるだけで、自ら計画を立て、道具を使って実行するエージェントの世界へ踏み込みます。

Session 06
Autonomy

「指示待ち」から「自律型」へ
— AIエージェントの仕組み

目的

人間の都度指示を必要とせず、大枠の「ゴール」に向けて自ら思考し、道具を使い、行動する「エージェント」の概念を理解する。

ReActの脳内
Thought
思考
「まず〇〇を検索しよう」と次の一手を考える。
Action
行動
道具（Web検索、計算機、DBなど）を呼び出して実行。
Observation
観察
結果を確認し、次のThoughtへループ。
構成案
AIエージェントとは
「手順」を一つずつ教えなくても、自分で「計画」を立てて実行するAI。
ツールの利用（Tool Use / Function Calling）
AIが自らの判断で、Webブラウザ、計算機、社内データベースなどの「道具」を使う仕組み。
これまでのAIとの違い
「対話相手・作業者」から「自律する部下」へ。
具体例
エージェントの挙動シミュレーション
「競合A社・B社の最新決算を比較サマリーして」
→ A社のIRページをWeb検索
→ PDFを読み込む
→ B社のIRページを検索
→ PDFを読み込む
→ 内容を比較計算する
→ 表形式で出力する
一連のプロセスを、追加プロンプトなしで自律的に完結させる。

Autonomy

開発現場を変える「コーディングエージェント」
— 実務への恩恵

目的

プログラミング未経験の現場担当者でも、自身の業務を効率化するツールを「自然言語」で内製できる（開発の民主化）時代を実感する。

構成案
「コード生成」から「自律開発」へ
人間がコードをコピペする時代から、エージェントが自らPC上でコードを書き、実行し、テストする時代へ。
AIが自らエラーを直す（自律デバッグ）
実行してエラーが出たら、AI自身がエラー文を読み取り「あ、このライブラリが足りないですね」と自ら修正して完成まで導く。
実務への恩恵
ノーコード／ローコードの究極形。非エンジニアが「言葉」でツールを作る。
実演シナリオ
タスク
100枚の現場写真に透かしを入れる
「このフォルダにある100枚の現場写真すべてに、右下に『2024年工事完了』の透かし文字を入れて、別フォルダに保存するPythonスクリプトを作って実行して」
エージェントが自らスクリプトを書き、試しに実行し、フォントエラーが出たら自己修正し、タスクを完遂する様子を紹介。

Session 08
Team

複数のAIが協働する「エージェントチーム」
— 人事評価シミュレーション

目的

異なる役割を持った複数のAIが議論・協働する「マルチエージェント」の概念と、それによる意思決定の質的向上を学ぶ。

構成案
マルチエージェントシステムとは
万能な1つのAIより、専門特化した（役割を与えられた）複数のAIのチームのほうが、ハルシネーションが減り、強固な成果を出せる理由。
AIの壁打ち・ボードメンバー化
経営会議や企画会議に「批判的思考を持つAI」や「顧客視点を持つAI」を同席させ、人間の盲点を補う使い方。
ケーススタディ
人事評価の多角化・合議
課長一人の視点による偏りやバイアスを補正する
課長（人間）が対象者の実績を入力。
Agent A ／ 次長ペルソナ 「短期的な売上は良いが、後進育成などのコンピテンシー評価が甘くないか？」
Agent B ／ 人事部長ペルソナ 「来期の全社戦略を考慮すると、チーム全体の底上げ貢献度も評価軸に入れるべきだ」
AI同士が意見をぶつけ合い、最終的に人間の評価者に「多角的な再評価の材料（レポート）」を提示する。
キーワード
マルチエージェント ペルソナ ディスカッション 意思決定支援 認知バイアスの補正
Session 09
Team

組織のAI化実践
— 専門業務プロセスをAIチームに委譲する

目的

本研修の集大成。複雑な専門業務プロセス全体を、AIエージェントたちの「バケツリレー」によって自動化するアーキテクチャ設計を学ぶ。

設計思想

複雑な業務を「単一のタスク」に分解し、それぞれに特化エージェントを配置する。人間は「承認」ボタンを押すだけ。数日かかっていた各部署間の確認作業が、AIのパイプライン化により数分で完了する。

ケーススタディ
仕様書受領 〜 受注可否 〜 システム自動入力フロー
Phase 1
解析（要件抽出エージェント）
顧客からの非定型な仕様書PDFから「検査項目A」「納期: 10/1」などの要件を自動抽出。
Phase 2
照合（規格・法令確認エージェント）
RAGを用いて、抽出された「検査項目A」と最新のJIS規格や法令・規制値を自動で照らし合わせ、「法的に問題なし」を判定。
Phase 3
判定（キャパシティ判定エージェント）
社内データベース（稼働状況）を参照し、「現在のラボのリソースで10/1納期での受注が可能か」を判定。
Phase 4
実行（システム入力エージェント）
受注可能と判断された場合、基幹システムへ必要情報を自動で起票（ドラフト作成）。
Human
承認
人間は最終的に「承認」ボタンを押すだけ。

Session 10
Governance

AI時代のガバナンスと、
人間が担う「ディレクター」としての新たな役割

目的

AIを安全かつ倫理的に運用するためのリスク管理を徹底し、「AIを使役する人間」に求められる新たなスキルとマインドセットを定義して研修を総括する。

構成案
AI活用のリスクとガバナンス
情報漏洩（シャドーAIの危険性）、著作権問題、ハルシネーションによる誤った意思決定の恐怖。
Human in the Loop（人間参加型）の原則
プロセスの完全自動化の危険性。AIの出力を鵜呑みにせず、重要な判断（法令確認、人事評価、契約書の押印など）には必ず「人間」が介在し、最終的な責任を負う仕組みの必須性。
「作業者」から「AIディレクター」へ
人間の役割は「自ら手を動かす」から、「業務プロセスを設計し、AIチームに適切な指示を出し、結果の品質を評価・責任を持つ」へとパラダイムシフトする。
研修の総括
アクションプランの策定。明日から各自が「自分の業務フローをどう分解し、どこをAIチームに任せられるか」を考えるワーク。
教訓
事例紹介
最終確認は、必ず人間が。
AIに契約書を作成させ、そのまま送付して法的トラブルになった事例などを紹介し、Human in the Loop の重要性を説く。
キーワード

Action Plan

AIは目的ではなく手段です。組織として何を達成したいのか、何を維持したいのかを軸に、現場発の活用案を持ち寄りましょう。
