Session 02 ／ LLMの仕組み

LLMとは何か
言葉を確率で紡ぐ機械。

Large Language Model（大規模言語モデル）。インターネット中の膨大な文章を読み込み、「次に来る確率が高い単語」を一語ずつ予測しているだけの、巨大な確率モデル。

Concept
In one sentence

世界中の文章を読みきった確率の達人。
"次に来そうな言葉" を、ひたすら一語ずつ書き続ける。

Diagram 01

仕組み — 入力から出力まで

LLMの中身は驚くほどシンプルな繰り返し処理。テキストを「トークン」に分割し、文脈を踏まえて次のトークンを予測する、これを一語ずつ繰り返している。

Step 01 ／ 入力
「今日の会議の議事録を3行で要約して」
Step 02 ／ トークン化
今日の会議の 議事録を3 行で要約して

文章をトークン（単語の断片）に分解する。日本語1文字＝1トークンとは限らない。

Step 03 ／ 文脈処理
Transformer Layers × N
各トークンが「どの単語と関係が深いか」を計算（Attention機構）
Step 04 ／ 次のトークン予測
議事録
72%
本日
14%
会議
8%
Step 05 ／ 繰り返し
最も確率の高いトークンを選び、それを入力に加えて Step 03 へ戻る → 一文章ができあがるまでループ。
Diagram 02

「Large」の意味 — 学習データと パラメータ数

GPT-4クラスのLLMは、人類が一生かけても読み切れない量の文章を学習している。これがLLMの「常識」の源泉。

人間の生涯読書量
≒1
Wikipedia 全文
≒4
大規模LLMの学習データ
≒80,000+

数兆トークン規模の文章を学習し、数千億〜数兆個のパラメータ（重み）に圧縮している。

LLMは "意味" を理解していない。
"統計的に尤もらしい次の言葉" を選び続けているだけ。
でも、それで多くの仕事ができてしまう。
Implication

「確率モデル」という理解が、業務利用の前提

LLMの正体を「魔法」だと思っていると、間違った使い方をする。「確率モデル」だと知っていれば、得意な仕事・任せてはいけない仕事の境目が見えてくる。

01

意味より「言葉のパターン」

LLMは「会議の議事録」という単語列の後に「を要約します」が来やすい、というパターンを知っている。だから議事要約が得意。ただし「会議の事実関係」を知っているわけではない。

02

同じ入力でも、出力は揺れる

確率モデルだから、毎回まったく同じ答えにはならない。これは欠陥ではなく仕様。再現性が必要な仕事には、別の工夫（テンプレート化や検証）が要る。

03

「学習に含まれること」しか答えられない

学習時点より新しいニュース、社内固有の情報、機密データはLLM単体では知らない。これを補うのが RAG や Tool Use といった仕組み。

Plain Language

用語解説 — むずかしい言葉をやさしく

このページに出てくる専門用語を、業務や日常に置き換えて理解する。

LLM Large Language Model

インターネット上の膨大な文章を読み込んで、「次に来そうな言葉」を当てる訓練を受けた巨大な確率モデル。ChatGPTやClaudeの中身は全部これ。

トークン Token

AIにとっての言葉の最小単位。単語より細かい "かけら" のこと。「学習」=「学」+「習」のように、AIは文字より少し大きい単位で文を切る。

Transformer Transformer

LLMの心臓部にあたる構造。文中の単語同士の関係（誰が誰を修飾するか、など）を一気に捉える仕組み。長文を読むときに「あの "彼" は誰のことだっけ？」と前後を確認しながら読む、賢い読み方。

Attention Attention

「いま注目すべきはどの単語か」を計算するTransformerの中核機能。会議中、議題に関係する人の発言だけ集中して聞き取る能力。

パラメータ Parameter

学習で自動調整された、LLMの内部のつまみ。数千億〜数兆個もある。巨大なオーケストラの楽器の数。1つ1つの音量バランスで全体のサウンドが決まる。

確率モデル Probabilistic Model

「絶対そう」ではなく「たぶんこれ」で動く仕組み。同じ質問でも答えが微妙に揺れる原因。天気予報のようなもの。「降水確率70%」とは言えても「絶対雨」とは言わない。
