AIエージェントとは
"手順" ではなく "ゴール" で動く。

一問一答のAIから、自分で計画を立てて行動するAIへ。「思考→行動→観察」のループを回し、ゴール達成まで自律的にタスクを進める。これがAIエージェント。

Concept
In one sentence

"手順を教える部下" ではなく、"目的だけ伝える部下"。
あとは自分で考え、調べ、進めてくる。

Reasoning（思考）と Acting（行動）の頭文字を取った設計パターン。AIが「考える→道具を使う→結果を見る→また考える」を繰り返す。

ゴールに到達するまで、このループを何度も回す。1ターンで終わらない、複雑なタスクをこなせるようになる。

Diagram 02

実行ログ — エージェントは内部でこう動いている

「競合A社とB社の最新決算を比較してまとめて」というゴールを与えたときの、エージェント内部の "つぶやき"。これが ReAct ループの実体。

[Goal] 競合A社とB社の最新決算を比較してまとめて

[Thought] A社のIRページから最新決算を探そう
[Action] web_search("A社 IR 決算")
[Observation] 検索結果から最新決算PDFのURLを取得

[Thought] PDFを読み込んで売上と営業利益を抽出する
[Action] fetch_pdf(url=A社IR/2024Q4.pdf)
[Observation] 売上: 8,500億 / 営業利益: 720億

[Thought] 同じくB社の決算も取得
[Action] web_search("B社 IR 決算")
[Observation] B社決算PDFのURLを取得 → fetch
[Observation] 売上: 6,200億 / 営業利益: 480億

[Thought] 2社の数字が揃った。前年比成長率も計算しよう
[Action] calculator(...)
[Observation] A社: 売上+8% / B社: 売上+12%

[Thought] 比較サマリーを表形式で出力する
[Action] finish(output="比較表 ...")
[Done] ゴール達成
人間が "手順" を教えなくても、AIが 自分で計画を立てる。
これがAIエージェントの本質。
Foundation

エージェントを成立させる3要素

単なる「会話AI」とエージェントを分ける、3つの構成要素。

01

計画立案能力（Planning）

ゴールから逆算して、ステップを自分で組み立てる。「まずA、次にB、その結果次第でCかD」と分岐も含めて考える。LLMの推論能力がこれを担う。

02

道具を使う能力（Tool Use）

Web検索、計算機、データベース、外部APIなど、必要な「道具」を自分で選んで呼び出す。これがあるから "知らない情報" にもアクセスできる。

03

記憶と自己修正（Memory & Reflection）

過去の試行と失敗を踏まえて、次の行動を変える。「さっきはこの検索ワードでダメだった→言い換えてみよう」というメタ的な振り返り。

Plain Language

用語解説 — むずかしい言葉をやさしく

このページに出てくる専門用語を、業務や日常に置き換えて理解する。

AIエージェント AI Agent

指示を待たず、ゴールに向けて自分で考え・動くAI。"出張の準備しといて" で完結する自律型のアシスタント。

自律性 Autonomy

次に何をするかをAI自身が決められる度合い。"手順を1から指示しないと動かない派遣" ⇔ "目的だけ伝えればやる正社員" のグラデーション。

ReAct Reason + Act

"考える(Reason)" と "行動する(Act)" を交互に繰り返すエージェントの基本ループ。"次は〇〇を調べよう" → 検索する → "次はこれをまとめよう" のリズム。

タスク分解 Task Decomposition

大きな依頼を、自分で小さな手順に分けること。"競合分析して" → "①A社調査、②B社調査、③比較表作成" にバラす力。

計画立案 Planning

実行前に "段取り" を立てる能力。料理する前に "材料切る → 炒める → 煮込む" を頭の中で組み立てる、あれ。
