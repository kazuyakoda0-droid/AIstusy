Session 07 ／ コーディングエージェント

コード生成 → 自律開発
AIが、自分でPCを操作して開発する。

ChatGPTにコードを書かせて人間がコピペする時代は終わり。コーディングエージェントは、エディタ／ターミナル／ファイルシステムを自分で操作し、コードを書き、実行し、テストする。

In one sentence

"コードを書くAI" から、"開発するAI" へ。
人間は、要件を伝えて成果物を確認する。

Diagram 01

2世代の開発スタイル

同じ「AIにコードを頼む」でも、人間の関わり方が一変する。コピペ職人 → ディレクター。

Before ／ 2023

コード生成

ChatGPTに「Pythonで○○を書いて」と頼む → 生成されたコードを人間がコピペ → 自分のエディタで実行 → エラーが出たらまた質問。

人間：仕様を伝える AI：コード生成 人間：コピペ＆実行 人間：エラーをAIに貼る 人間：修正コードをコピペ ...繰り返し
→
Paradigm Shift
Now ／ 2024〜

自律開発

エージェントに「○○というツールが欲しい」と伝える → エージェントが自分でファイルを作り、コードを書き、実行し、エラーを直し、完成させる。

人間：要件を伝える AI：ファイル作成・コード記述 AI：実行 → エラー検知 AI：自己修正 → 再実行 AI：成功 → 報告 人間：成果物の確認
Diagram 02

エージェントが実際に動く様子

「100枚の現場写真に透かしを入れる」というタスクを与えたときの、ターミナル上での動作イメージ。

~/projects/photo-task
$ claude "現場写真フォルダの全ファイルに『2024年工事完了』の透かしを入れて別フォルダに保存して"

● ① 入力フォルダのファイル一覧を取得
● ② Pythonスクリプトを作成
● ③ 実行してテスト → 全体実行

● watermark.py を作成中...
✓ watermark.py 作成完了 (32 lines)

● 1枚でテスト実行中...

● Pillow ライブラリが必要なようです。インストールします。
$ pip install Pillow
✓ インストール成功。再実行します。

● 1枚テスト → 出力確認 → OK

✓ 完了：100枚すべてに透かしを入れ、output/ フォルダに保存しました。

エージェントは「コードを書く → 実行 → エラー検知 → 自己修正」のループを、人間の介入なしで回す。これが 自律デバッグ。

プログラミングは、"言葉で指示する仕事" になる。
非エンジニアでも、ツールを内製できる。
Tools

主要なコーディングエージェント

2025年時点の代表例。それぞれ得意領域が違うが、共通するのは「自然言語 → 自律的にコードを書いて実行」というスタイル。

01

Claude Code

ターミナルベースで動くコーディングエージェント。プロジェクトディレクトリ全体を理解し、複数ファイルにわたる変更、テスト実行、Gitコミットまで自律的に行う。

02

Cursor

VS Code 風のエディタにAIが統合されたIDE。コード補完、エージェント機能、プロジェクト全体への適用などを直感的なUIで操作できる。

03

GitHub Copilot / Copilot Workspace

GitHub標準のAI開発支援。Issue起票からプルリクエスト作成まで、開発フローに沿った自律支援。
