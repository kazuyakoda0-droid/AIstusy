Session 02 ／ プロンプト設計

プロンプト4要素
構造化された指示の作法。

AIは「超優秀だが融通の利かない新入社員」。曖昧な丸投げは曖昧な答えを返す。Role / Context / Task / Format の4つを書き分けるだけで、出力の精度は劇的に変わる。

Concept
In one sentence

新入社員に仕事を頼むときと、同じ作法で。
「誰として」「どんな状況で」「何を」「どんな形で」。

Diagram 01

4つの要素

どんなプロンプトもこの4要素に分解できる。書き始める前に「自分は今、どの要素が抜けているか」を確認するだけで質が変わる。

Element 01

役割

Role

AIに「誰として答えてほしいか」を伝える。専門家の口調、視点、語彙が引き出される。

Example あなたはベテランのコピーライターです。
Element 02

文脈・背景

Context

状況、対象、制約条件など、判断の前提を共有する。これがないと一般論しか返ってこない。

Example 20代女性向けの時短スキンケアコスメを発売します。価格帯は3,000円前後。
Element 03

タスク

Task

具体的に何をやってほしいか、動詞で明示する。「考えて」「作って」「比べて」など。

Example 商品の魅力を伝えるキャッチコピーを提案してください。
Element 04

出力形式

Format

箇条書き／表／JSON／文字数制限など、欲しい形を指定する。後工程の手戻りが激減する。

Example 箇条書きで5つ、それぞれ解説付きで。1案30文字以内。
Diagram 02

4要素をマーキングしてみる

同じ依頼を、4要素を意識して書き直すとこうなる。色付き部分が各要素。書く順番は自由。

あなたはベテランのコピーライターです。 20代女性向けの時短スキンケアコスメを発売します。 商品の魅力を伝えるキャッチコピーを提案してください。 箇条書きで5つ、それぞれ解説付きで。
Before / After

同じ依頼でも、結果がこう変わる

「丸投げ」と「構造化」、両方で同じ業務を依頼したときの出力品質の差。

Bad ／ 丸投げ
新商品のキャッチコピー考えて
AIの返答：「キラリ☆あなたを輝かせるコスメ」「毎日に小さなご褒美を」… 抽象的、誰向けかも分からない、刺さらない。
Good ／ 構造化
役割 + 文脈 + タスク + 形式
AIの返答：「忙しい朝こそ、3分美容。」「鏡の前で迷う時間、終わり。」… ターゲットの行動・心理に刺さるコピー、5案、選びやすい。
プロンプトは "祈り" ではなく "設計"。
仕事を任せるときと、まったく同じ作法。
Templates

明日から使える、業務テンプレ

4要素を最初は型として埋めてみる。慣れてくると自然に書けるようになる。

01

メール返信のドラフト

役割：あなたは社外向けの丁寧なビジネスメールを書く担当者です。
文脈：取引先からの値引き要請に、応じられない旨を伝えたい。関係は維持したい。
タスク：返信メールのドラフトを作成して。
形式：件名、本文。本文は400字以内、結論先出し。

02

議事録の要約

役割：あなたは経験豊富な議事録作成者です。
文脈：添付は1時間の会議の文字起こし。出席者はA・B・C。
タスク：決定事項・未決事項・Next Actionを抽出して。
形式：3カラムの表。Next Actionには「誰が・いつまでに」を必ず含む。

03

企画の壁打ち

役割：あなたは批判的視点を持つマーケティング部長です。
文脈：添付は私の新企画書。ターゲット曖昧で悩んでいる。
タスク：ターゲット設定の弱点を指摘し、絞り込むための質問を3つして。
形式：指摘 → 質問 → 私の回答欄、を3セット。

Plain Language

用語解説 — むずかしい言葉をやさしく

このページに出てくる専門用語を、業務や日常に置き換えて理解する。

プロンプト Prompt

AIへの指示文・お願いごとのこと。新人にメモで仕事を渡すときの、そのメモの中身。

プロンプトエンジニアリング Prompt Engineering

指示の書き方を工夫して、AIの出力品質を上げる技術。部下への指示の出し方の上手・下手と同じこと。

ゼロショット Zero-shot

例を見せずに、いきなりタスクをお願いするやり方。お手本なしで「キャッチコピー10個ください」と頼む。

フューショット Few-shot

「こういう感じで」と1〜数個の例を見せてからお願いするやり方。"先輩の過去資料3つ見せて、同じトーンで作って" と頼む方が精度が上がる。

Chain of Thought CoT

「順番に考えて」と一言加えるだけで、AIの推論力が上がる現象を使ったテクニック。計算問題で「途中式を書いて」と言う、あの感覚。

ロール／役割 Role

「あなたはベテランの〇〇です」と立場を与える指示。トーンや視点が一気に変わる。同じ質問でも、相手を弁護士に頼むか、デザイナーに頼むかで答えが変わるのと同じ。
