ツールの利用
AIに、道具を持たせる。

Function Calling とも呼ばれる、エージェントの中核機能。AIに「Web検索」「計算機」「データベース照会」などの道具を渡し、必要なときに自分で選んで使わせる。

In one sentence

AIに "知っていること" だけで答えさせない。
必要な道具を渡して、"調べさせる"。

Diagram 02

ツールの "定義" は、こう渡す

技術的には、AIにツールを使わせるためにツールの仕様（名前・引数・説明）を渡すだけ。AIは説明文を読んで「いつ呼ぶか」を自分で判断する。

// AIに渡すツール定義（JSON Schema）
{
  "name": "query_inventory",
  "description": "社内倉庫の在庫状況を商品コードで照会する",
  "parameters": {
    "product_code": { "type": "string", "description": "7桁の商品コード" },
    "warehouse": { "type": "string", "description": "東京/大阪/福岡" }
  }
}

// AIがこの定義を読んで、ユーザーの質問に応じて呼び出す
// User: 「商品AB12345、東京倉庫にどれくらいある？」
// AI: query_inventory(product_code="AB12345", warehouse="東京")

肝は「説明文」。AIは "description" を読んで、ツールの使いどころを判断する。だから、ツールに渡す説明文はプロンプト設計と同じくらい重要。

AIに "全部知らせる" 必要はない。
"調べ方を知っている" AIに、必要なときに調べさせる。
Use cases

ツール利用が変える業務

「LLMだけでは無理」と思っていた業務が、ツールを揃えるだけで一気に手の届く範囲に入る。

01

正確な計算が必要な仕事

見積金額の集計、利益率計算、複雑な数式評価。AIに直接計算させず、calculator ツールに任せる。「数字に強いAI」が実現する。

02

最新情報を踏まえた回答

「今週の競合の動向は？」「最新の法改正は？」 web_search ツールを使わせ、ソース付きで回答。ハルシネーションが激減する。

03

社内システムを操作する

「Aさんの来週の予定を空けて」「○○案件のステータスを更新して」など、AIがカレンダー・CRM・基幹システムを直接操作する。アクション型AIへの道。

04

コードを書いて実行する

run_python ツールを使わせると、「CSVを読み込んで集計したグラフを描いて」のような依頼に、自分でコードを書いて実行する。次回の「コーディングエージェント」につながる流れ。
